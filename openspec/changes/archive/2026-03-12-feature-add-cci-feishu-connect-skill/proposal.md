# 提案：新增CCI飞书连接Skill

## 背景
为了配合cci项目的飞书桥接功能，需要在cckit的skills/cci目录下新增`/connect-feishu`指令，实现Claude CLI与飞书桥接程序的联动。

## 核心功能
1. 进程冲突检测：调用指令时检查是否有正在运行的桥接进程，有则提示用户先断开
2. 启动桥接程序：无冲突时自动后台启动飞书桥接程序，绑定当前CLI会话
3. 注册断开指令：注册`/disconnect`指令，支持用户手动断开桥接连接
4. 自动清理：桥接程序退出时自动清理临时文件和锁

## 文件位置
- Skill文件路径：`/opt/dev/apps/cckit/skills/cci/connect-feishu.ts`

## 交互流程
1. 用户在Claude CLI中输入 `/connect-feishu`
2. Skill检测本地是否有运行中的桥接进程：
   - 存在：提示"已有飞书连接正在运行，请先输入 /disconnect 关闭后重试"
   - 不存在：启动飞书桥接程序，后台运行
3. 桥接程序自动绑定当前CLI会话的stdin/stdout，实现双向消息同步
4. 用户输入 `/disconnect` 或在飞书发送"断开连接"，桥接程序退出，清理资源

## 交付标准
- 指令调用正常，冲突检测逻辑正确
- 桥接程序能正常启动和绑定当前CLI会话
- 断开指令能正常结束桥接进程并清理资源
